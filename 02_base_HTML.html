<form>, <legend>, <fieldset> et <button>

Pour générée un formulaire ont utilisent la balise <form>. Les <form> constitue un bon 80% en développement que ce soit pour une inscription ou pour crées des produits par exemple, leurs utilisations est courante, si il y as bien une balise à la quelle il est difficile de faire l'impasse c'est bien celle ci.

Un formulaire à sa propre bordure attitrée, pour sa il faut placer une balise <fieldset> à l'intérieur de <form>. Cette bordure est personnalisable via le CSS au besoin, elle ne constitue pas un élément majeur du formulaire et il peu fonctionner bien sans

Comme son nom l'indique, <legend> permet de placer une legende au formulaire, considérée la legende comme un titre appartenant au formulaire.

Un formulaire à comme utilité d'envoyez quelque chose, une donnée ou plusieurs jeu de donnée au serveur. Pour se faire il nous faut le <button> pour envoyez les données. Le bouton peu se placer à l'interieur du formulaire comme à l'extérieur selon vos besoins et il peut être remplacer par un <input> également.

Pour fonctionner correctement, un formulaire réclame certain paramètre selon vos projets.

<form action="/signup"> L'attribut action détermine où les informations vont être traitée, la valeur dépend de votre architecture de projet, si l'on part du principe que nous restons en JavaScript natif et que nous simulons l'envoit de donnée via une base de données factice, alors la valeur seras une page. Sous Laravel, framework PHP la valeur seras une route, route gérée par un middleware, et ce Middleware appelera une fonction derrière pour traité les données.

<form action="/signup" method="post"> L'attribut method réclame une valeur, soit get, soit post. Ces valeurs sont des méthodes d'envois, l'une fait transitez les données via l'url du navigateur (get) tandis que l'autre mélange le tout et l'envois directement au serveur pour son traitement (post). La valeur get est dépréciez car si les données sont visibles via l'url, alors elles peuvent être aisément modifié par n'importe qui. Autre problématique, la valeur get ne permet pas d'envoyez des données conséquentes, celle-ci sont bridée à 250 caractères max de souvenir. Ces valeurs sont des protocols HTTP, une subtilité qui aura sont importances plus tard mais pas ici. Gardez à d'esprits que GET transite les données via l'url, donc visible pour la partie cliente, et que post diffuse ces données via un corps cotée serveur non visible par le client (sauf avec un peu de connaissances et si les data ne sont pas hasher correctement).

<form action="/signup" method="post" enctype="multipart/form-data"> L'attribut enctype définit le format général des données. La valeur multipart/form-data spécifie que des données peuvent être aussi de format jpg, png, bitmap pour les images et même un ensemble de fichier.

Enfin dans le contexte que nous faisont un formulaire en JS, le <button> à besoin d'au moins une information, un ID. Cette ID pourras être utilisée cotée serveur pour enclencher une CallBack addEventListener.

<label> et <input>

<label> est le nom qui doit être associer à son <input>

<input> est un champ de formulaire, les champs de formulaire permette à l'utilisateur de saisir des informations principalement. Ces champs peuvent avoir plusieurs rôles, certain sont conçue pour acquérir des données en chaine de caractères, d'autre sont conçue pour les emails, les numeros de téléphones ou encore de mot de passes. Ce que la balise <input> traite est déterminé par son attribut type, cette balise à besoin également d'un autre attribut, l'id. L'Id permet de gérer la donnée cotée serveurs pour le cas du JavaScript, en php l'attribut qui repère la donnée est name. Enfin l'id doit être associé à un label.

L'attribut placeholder permet d'inscrire une définition à l'intérieur de <input>, cette définition s'efface automatiquement dès lors que l'utilisateur saisie une information dans le champ en question. Ceci n'est pas obligatoire mais, permet de guidée l'utilisateur dans la donnée qu'il doit saisir, il est donc conseillez de faire ceci, au cas échéant il faudra placer les balises comme ceci (voir ci-dessous)

<br> permet de faire un saut de ligne, déconseillez hors code sandbox passez plutôt via le CSS.

<select>, <optgroup> et <options>

<select> permet de faire des listes déroulantes, <optgroup> permet de gérée des groupes parmi cette liste déroulante et <options> constitue les valeurs appartenant à <optgroup>

Il n'est pas nécessaire de placer une liste déroulante dans un formulaire sauf si vous avez besoin de charger un jeu de données dynamiquement pour vos champs <options> comme montrer avec leurs valeurs.
